We are shipping the next milestone with a focus on reliability and a simpler onboarding path. The goal is to get new users to a successful first run faster without breaking existing integrations.

## Summary

The release tightens error handling in the agent harness, simplifies project setup, and keeps the public API stable for integrators who already depend on current behavior.

## Goals

- Reduce time-to-first-success for new users
- Keep the existing API stable for integrators
- Ship with observability hooks so we can catch regressions in the first week

## Timeline

- **Apr 1** — private beta with design partners
- **Apr 15** — general availability
